tap_multitap
Non-interpolating multitap delay unit
This object depends on the sample-rate being set correctly in order to set the delays, so be sure you don't overlook it.

Also, the constructor requires an argument to define the maximum delay time you will use (it allocates a memory buffer for the delay). If the argument is a float the argument will be interpreted in milliseconds, if the argument is a long it will be interpreted as a number of samples to allocate.

Attributes
    k_delay_ms The delay time for a tap defined in milliseconds. This requires two arguments: the tap number [0 - 99] and the delay time.
    k_delay_samples The delay time for a tap defined in samples. This requires two arguments: the tap number [0 - 99] and the delay time.
    k_gain The gain for a tap defined in decibels. This requires two arguments: the tap number [0 - 99] and the gain (where 0 dB = full scale, -6 dB = 0.5, etc.).
    k_numtaps The number of taps on the delay line. [1 - 100, default: 1]
    k_master_gain The master output gain for the multitap unit defined in decibels (where 0 dB = full scale, -6 dB = 0.5, etc.).

Note that all of the taps are mixed in this object before being output. You don't have access to the individual delay outputs.
