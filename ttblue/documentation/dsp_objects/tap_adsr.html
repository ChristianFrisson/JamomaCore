tap_adsr
Standard ADSR envelope generator
This object features two DSP loop. One sports a signal_vector input for sample-accurate triggering of the envelope. The other has no signal input, and you can trigger it (with vector-accuracy) using the method below.
Attributes
    k_attack The attack time specified in milliseconds.
    k_decay The decay time specified in milliseconds.
    k_sustain The sustain level specified in the normalized range (0.0 to 1.0).
    k_sustain_db The sustain level specified in decibels (where 0dB = full scale, -6dB = 0.5, etc.).
    k_release The release time specified in milliseconds.
    k_mode The shape of the envelope segments. The attribute should be set with one of the following selectors:
      k_mode_linear [default]
      k_mode_exponential
Methods
    set_trigger(bool value) Sending a 1 triggers the attack. Sending a 0 triggers the release. This will only work if you are using the first of the two DSP loops below
DSP Loop
void dsp_vector_calc(

)
If you use this dsp routine, you can trigger the envelope using the set_trigger() method above. The envelope is always generated at the sample-rate, but using this DSP loop the triggering of the envelope is done at the vector. This is probably adequate for most applications (and it is also faster).

Use the "in" sample_vector to pass the trigger to the envelope-generator with sample accuracy.
