tap_allpass
Allpass filter
You must specify the maximum delay time as an argument (in milliseconds) to the constructor. This allocates a buffer for the delay. This can only be changed by using the init() method described below.
Attributes
    k_delay The filter’s amount of delay specified in milliseconds. [default: 0 ms]
    k_gain The gain coefficient for the filter. You should keep this below 1.0 to prevent blow-ups. [default: 0.0]
Methods
    clear() Use to reset the buffers to zero. Handy in the event of something blowing-up.
    init(attribute_value ms) Use this if you need to resize the buffers for the delay. This frees and reallocated memory, so take care not to do it in a timing critical situation (such as in an interrupt).
