tap_upsample
Upsample a signal
This object is upsamples a signal. This is intended for applications where a block of objects might be run at a lower sample rate for efficiency, and then needs to be returned back up to the original sample-rate - though it may be useful in other situations as well. NOTE: the input vector will not be fully used unless the factor is set to "1" (no upsampling).
Attributes
    k_factor Sets the amount of upsampling performed on a signal. It does this by using the factor to determine how many samples to provide for each step in the vector. By default this is "1" which equates with no upsampling. A value of "2" is the same as doubling the sample rate (doubling the number of samples in a vector), a value of "4" is quadrupling the sr to 4x the original.

    As a consequence, the input vector will not be fullly utilized (the vector_size passed as an argument is used to determine the size of the output vector). For example, the upsampled input will take only the first half of the vector_size in the input if the factor is set 2. The upsampled input will only be used for the first 25% of the vector if the factor is set to 4. Be sure to pass the new (longer if you've upsampled) vector_size to the objects that follow this object to get predictable results.
