tap_delay
Delay unit with optional interpolation
This object depends on the sample-rate being set correctly in order to set the delay, so be sure you don't overlook it.

Also, the constructor requires an argument to define the maximum delay time you will use (it allocates a memory buffer for the delay). If the argument is a float the argument will be interpreted in milliseconds, if the argument is a long it will be interpreted as a number of samples to allocate.

Attributes
    k_delay_ms The delay time defined in milliseconds.
    k_delay_samples The delay time defined in samples.
    k_interpolation A selector which determines what type of interpolation to use, if any. The default is k_interpolation_linear.

A simple delay of the input to the output. Interpolation is determined by the above attribute.
void dsp_vector_calc(
    sample_vector in1,
    sample_vector in2,
    short vector_size,
    sample_vector out
)
In this dsp loop the in2 signal_vector is the delay time passed as a signal. Use this method if you want smooth pitch effects using the delay. The delay time should be specified in milliseconds.

Do not set the delay time attribute is using this DSP routine.
