Installing

  1. Place the Jamoma folder in your Max searchpath (such as in the Cycling '74 folder).
  2. If you are using Windows:
    • Three DLL files are required. They must be placed into the "support" folder in your Max application's folder. These three files are located deep in the Jamoma folder and must be moved:
      • iconv.dll - located in Jamoma\library\externals_3rd_party\libiconv
      • zlib1.dll - located in Jamoma\library\externals_3rd_party\libiconv
      • libxml2.dll - located in Jamoma\library\externals_3rd_party\libxml2\win32\bin
    • If you have installed previous versions of Jamoma, please check if there is a file named jamoma-init.txt in your init folder (inside of the Cycling '74 folder). If so, delete this.
  3. If you are using Mac:
    • If you checked Jamoma out using SVN (rather than downloading a release from the website):
      • Unzip (double-click) the build_mac.zip archive located in the Jamoma/library/externals folder.
      • Unzip any archives located inside the various subfolders of the Jamoma/library/externals_3rd_party folder.
    • If you downloaded a release from the website, ignore the steps above.
  4. Move the jcom.meter~ external (in /library/externals/build_mac or /library/externals/build_win depending on your platform) into your max-startup folder (inside of the Cycling '74 folder)
  5. Check out the Jamoma documentation.

Optional customization you may want to do for your install might include the following:

  1. The overview patch (Jamoma/documentation/jamoma-overview.mxt) may be copied to the max-startup folder or in Max's patches/extras folder.
  2. The templates (patches located in Jamoma/documentation/templates) are most effective when copied to Max's patches/templates folder.

The Jamoma distribution depend on certain 3rd party externals. For simplicity we have chosen to include these externals as part of the distribution. If you already have installed some or all of these externals, make sure that that the Jamoma distribution do not introduce older versions of the externals than you have already. All such externals can be found in the Jamoma/library/externals_3rd_party folder.

Requirements

  • Jamoma requires Cycling '74's Max version 4.6 or newer (Mac) or version 4.5.7 or newer (Windows).
  • Audio modules require Cycling '74's MSP.
  • Video modules require Cycling '74's Jitter (version 1.5.2 or higher).
  • Some ambisonic modules require both Cycling '74's MSP and Jitter.

Examples

To get started, there is an examples folder in the Jamoma/documentation folder. Trond Lossius has also created some excellent tutorials. Also, if you are interested in creating your own modules, be sure to check out the JIG (Jamoma Interface Guidelines) which is in the documentation folder.

Known Issues

The following are the problems of which we are currently aware:

  • When a patch is first drawn, the modules may appear block. Refreshing the screen by sending the ;max refresh message or unlocking the patch will resolve the problem.
