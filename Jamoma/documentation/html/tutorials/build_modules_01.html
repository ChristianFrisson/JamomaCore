Tutorial 1: Introduction

Jamoma is a proposition for a standard set of guidelines for development of modules in Max. Jamoma consists of two parts: A recommendation and an implementation of those recommendations. Jamoma offers a numebr of pssoble benefits:

  • Jamoma encourage reuse of modules from one project to the next.
  • Jamoma can simplify development in projects involving several artists/developers by offering a common protocol for modules.
  • Jamoma might simplify the creation of clear, flexible and screen-efficient GUI.
  • Jamoma offers a simple and flexible solution for remote control of the states of parameters in the various patches, including cue handling.
  • As Jamoma integrate the OpenSoundControl protocol, it is relatively straight-forward to set up a network of several computers running MaxMSP/Jamoma with comunication between the various computers.
  • We hope that Jamoma might simplify exchange of patches and modules between Max users in a similar way to how 3rd party externals are currently being made available.

Jamoma modules can roughly be subdivided into 3 categories, control, audio and video, in a similar way to the division between Max, MSP and Jitter. Of course you can imagine modules dealing with audio and video, but let's stick to these three categories for the time being.
