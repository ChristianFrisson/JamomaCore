Parameters

Name Type Range Clipmode Ramp Enabled Repetitions Description
/disable_ui_updates toggle default none no yes Turn off the updating of user interface elements when parameters change. This may be done to conserve CPU resources.
/set_clock msg_symbol default none no yes Sync module to external clock.
/set_clock max set module to use the Max scheduler (default behaviour).

 

Messages

Name Type Range Clipmode Ramp Enabled Repetitions Description
/clear generic default both no yes Reset by removing all existing mappings.
/create generic default both no yes Create a new mapping. Arguments are: [mapping_name] [mapping_source (OSC)] [optional algorithm to apply][ ->][mapping_destination (OSC)].
/get_parameter_names n/a n/a both no yes Returns a dump of symbols out the feedback outlet with the names of parameters accessible in this module.
/help generic n/a both no yes Open the online html reference for this module.
/load_mappings generic default both no yes Load mappings from file.
NOTE: jmod.mappings will be watching the file and automatically update if the content of the file is changed. This way you can use an external editor to work on the cue script while Jamoma is running.
/load_settings msg_symbol n/a both no yes Open an xml-preset file and recall the first preset in that file. An optional argument defines the file to open.
/modify generic default both no yes Remove an existing mapping. the required argument is the mapping_name.
/preset_recall msg_int default both no yes Recall a preset by number - you can also choose presets from the module menu.
/remove generic default both no yes Remove an existing mapping. the required argument is the mapping_name.
/restore_defaults n/a n/a both no yes Open the default preset file and recall the first preset in that file.
/save_mappings generic default both no yes Save mappings to a file. An argument determines the file path and name. If no argument is given then a file dialog will be presented.
/save_settings msg_symbol n/a both no yes Write an xml-preset file to disk. An optional argument defines the file to open.
/update generic default both no yes If you have made changes in the viewer (or an external text editor) then you will need to send this /update message in order for them to take effect. This done automatically when you open a fresh file.
/view generic default both no yes View current mappings in a text window.
/view_internals n/a n/a both no yes Attempts to open the internal algorithm for viewing. This works for most modules. Some modules may choose to cloak the algorithms - preventing this message from functioning.

 

About Jamoma

Jamoma is a system for creating and exchanging structured Max patches. It consists of both a set of guidelines and an implementation of those guidelines. For more information please visit jamoma.org.
