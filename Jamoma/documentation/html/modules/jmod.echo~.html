jmod.echo~

Stereo echo module.

Table of Contents | Index of Modules | Jamoma.org

Configuration

Module Type: audio
Interface Size: 1U-half

Number of signal inlets: 2
Number of signal outlets: 2

Parameters

Name Type Range Clipmode Ramp Enabled Repetitions Description
/audio/bypass toggle default none no yes When active, this attribute bypasses the module's processing algorithm, letting audio pass through unaffected.
/audio/gain/midi msg_float 0 158 both yes yes Set gain as MIDI value.
/audio/meters/defeat toggle default none no yes When active, this attribute turns off the signal level meters in the module to conserve CPU.
/audio/mix msg_float 0 100 none yes yes Controls the wet/dry mix of the module's processing routine in percent.
/audio/mute toggle default none no yes When active, this attribute turns off the module's processing algorithm to save CPU
/audio/sample_rate msg_int -2 0 none no no Sets the relative sample-rate for this module.
/cf msg_float 30.000 11025.000 both yes no Center frequency (Hz)
/clock/set msg_symbol default none no yes Sync module to external clock.
/set_clock max set module to use the Max scheduler (default behaviour).
/feedback msg_float 0.000 1.000 both yes no "Feedback."
/filtergain msg_float -24 24 both yes no Gain (dB)
/filtertype menu default none no yes What kind of filter to use. Possible values: lowpass | highpass | bandpass | bandstop | peaknotch | lowshelf | highshelf
/left msg_int 0 10000 both yes no Delay time in milliseconds for left channel.
/q msg_float 0.000 100.000 both yes no Resonance (Q)
/right msg_int 0 10000 both yes no Delay time in milliseconds for right channel.
/ui/freeze toggle default none no yes Turn off the updating of user interface elements when parameters change. This may be done to conserve CPU resources.
