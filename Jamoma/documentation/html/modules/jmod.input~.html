jmod.input~

A generic modul for audio input

Table of Contents | Index of Modules | Jamoma.org

Configuration

Module Type: audio.no_panel
Interface Size: 2U-half

Number of signal inlets: 0
Number of signal outlets: 2

Parameters

Name Type Range Clipmode Ramp Enabled Repetitions Description
/audio/gain/midi msg_float 0 158 both yes yes Set master gain as MIDI value.
/ch_1 msg_int default none no yes Set logical input for left channel.
/ch_2 msg_int default none no yes Set logical input for right channel.
/clock/set msg_symbol default none no yes Sync module to external clock.
/set_clock max set module to use the Max scheduler (default behaviour).
/gain_1/midi msg_float 0 158 both yes yes Set left channel gain as MIDI value.
/gain_2/midi msg_float 0 158 both yes yes Set right channel gain as MIDI value.
/link toggle default none no yes Link gain levels for left and right channels.
/loop toggle default none no yes Loop soundfile.
/play generic default none no yes Play sound file.
/source_select menu default none no yes Select what audio source to use.
/ui/freeze toggle default none no yes Turn off the updating of user interface elements when parameters change. This may be done to conserve CPU resources.
/ui/refresh msg_bang default none no yes Update displayed values for module to reflect current state.

 

Messages

Name Type Range Clipmode Ramp Enabled Repetitions Description
/audio/gain msg_float -144 0 none yes yes Set master gain as dB value.
/audio/gain/dec generic N/A none no yes Increase master gain MIDI value by one.
/audio/gain/inc generic N/A none no yes Decrease master gain MIDI value by one.
/direct generic default none no yes container for various messages sent directly to the sfplay~ object. Refer to the MSP documentation for further details.
/documentation/help generic n/a none no yes Open the help patch for the module.
/documentation/html generic n/a none no yes Open the online html reference for this module.
/dsp_status generic default none no yes Open DSP Status window.
/gain_1 msg_float -144 0 none yes yes Set left channel gain as dB value.
/gain_1/dec generic N/A none no yes Increase left channel gain MIDI value by one.
/gain_1/inc generic N/A none no yes Decrease left channel gain MIDI value by one.
/gain_2 msg_float -144 0 none yes yes Set right channel gain as dB value.
/gain_2/dec generic N/A none no yes Increase right channel gain MIDI value by one.
/gain_2/inc generic N/A none no yes Decrease right channel gain MIDI value by one.
/module/parameter_names/get n/a n/a none no yes Returns a dump of symbols out the feedback outlet with the names of parameters accessible in this module.
/module/size/get generic default none no yes Get size of module.
/module/view_internals n/a n/a none no yes Attempts to open the internal algorithm for viewing. This works for most modules. Some modules may choose to cloak the algorithms - preventing this message from functioning.
/preset/default n/a n/a none no yes Open the default preset file and recall the first preset in that file.
/preset/direct generic n/a none no yes Access directly to the pattrstorage object. See Max Reference.
/preset/load msg_symbol n/a none no yes Open an xml-preset file and recall the first preset in that file. An optional argument defines the file to open.
/preset/recall msg_int default none no yes Recall a preset by number - you can also choose presets from the module menu.
/preset/save msg_symbol n/a none no yes Write an xml-preset file to disk. An optional argument defines the file to open.
/preset/store msg_int default none no yes Store a preset by number in memory. All presets present in memory will be written to disk when you send a save_settings message to the module.
/sf_open msg_symbol default none no yes Open sound file specified by filename. Without a filename a standard open file dialog allows you to choose a file.

 

Return

Name Type Range Clipmode Ramp Enabled Repetitions Description
/module/size generic default none N/A yes Size of module.
/sf_report generic default none N/A yes Report when sound file reach end point and loop to start point.

 

About Jamoma

Jamoma is a system for creating and exchanging structured Max patches. It consists of both a set of guidelines and an implementation of those guidelines. For more information please visit jamoma.org.
