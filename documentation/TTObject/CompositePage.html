ttantidenormal

process floats to ensure that they are not denormals 

double ttantidenormal(
    double value);
Discussion

It is important on Intel processors (though not on PPC processors) to filter out denormal values in performance critical code. The use of denormals is extraordinarily slow, and easy to cause in recursive algorithms such as audio filters. If TT_DISABLE_DENORMAL_FIX is #defined, then this function will pass all input through unfettered.

ttclip

limits the range of an input number between a low_bound and high_bound 

long ttclip(
    long value,
    long low_bound,
    long high_bound);
Parameter Descriptions
value
This is the input that will be clipped.
low_bound
Defines the lower limit to which the input should be clipped.
high_bound
Defines the upper limit to which the input should be clipped.
Discussion

All three args for the ttclip function (and the return value) must be of the same type. This method is optimized to not use branching, and thus should be relatively fast.

TTErr

The primary return type for TTBlue methods. 

typedef long TTErr; // used for returning error codes
Discussion

TTErr is typically an integrer value represented by the Error Codes enum.

TTSampleValue

A value representing a single audio sample. 

typedef float TTSampleValue;
Discussion

TTSampleValue should be used any place a sample value is what the value represents. This will enable us to change the type in the future if needed. For example, to use 64-bit floats.

#defines

TTOBJECT_BASE_CLASS

Define the TTBlue base class.

#define TTOBJECT_BASE_CLASS NSObject
Discussion

By default, the TTBlue base class (TTObject) is a subclass of NSObject. However, if TTOBJECT_BASE_CLASS is defined as something else, then that will be used as the base class instead.

TTOBJECT_BASE_CLASS_HEADER

Define the TTBlue base class header file name.

#define TTOBJECT_BASE_CLASS_HEADER TTOBJECT_BASE_CLASS_HEADER // This is silly, but it prevents the above HeaderDoc from barfing
Discussion

If TTBlue is defined as a subclass of an object other than the default NSObject, then it is likely that an appropriate header file for that base class will need to be included. If TTOBJECT_BASE_CLASS_HEADER is defined, then that the string that defines it will be included by the preprocessor before the TTObject class is set up.
