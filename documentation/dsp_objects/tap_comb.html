tap_comb
IIR comb filter with a lowpass filter in the feedback loop
You must specify the maximum delay time as an argument to the constructor. This allocates a buffer for the delay. This cannot be changed during the life of the comb filter object.
Attributes
    k_cutoff_frequency The lowpass filter’s cutoff frequency, specified in Hertz [0-nyquist].
    k_clip A clipping threshold for the comb filter's feedback. A value of 0 will turn off clipping. Other values will set the range, outside of which samples will be clipped. This is useful for preventing the filter from blowing up. The default is 1.
    k_delay The comb filter's delay time in milliseconds. This value should be lower than the maximum delay time (specified as an argument to the object's constructor).
    k_delay_static The comb filter's delay time in milliseconds. This value should be lower than the maximum delay time (specified as an argument to the object's constructor). Unlike, the k_delay attribute, setting the delay time with this will not update the decay time internally.
    k_decay The comb filter's decay time in seconds. This value controls the amount of feedback in the comb filter.
    k_feedback Directly accesses the comb filter's feedback coefficient. This may be used in place of the aforementioned k_decay attribute.
Methods
    clear() Use to reset the buffers other feedback storage. Handy in the event of something blowing-up.
