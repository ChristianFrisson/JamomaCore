tap_wavetable
Wavetable-based oscillator (with optional modulation)
This object features two DSP loops, one with a signal_vector input for modulating the frequency of the oscillator, the other with no signal input.
Attributes
    k_frequency The base frequency of the oscillator specified in Hertz [0.0 - Nyquist, default: 0.0 Hz].
    k_mode You can choose one of the built-in waveforms. The waveforms with the "_mod" tag are suitable for use as modulators, i.e. they are in the range of 0.0 to 1.0 instead of the usual audio range of -1.0 to 1.0. Setting this attribute is actually calling the fill() function of the member tap_buffer object this oscillator is built on. The attribute should be set with one of the following selectors:
      k_mode_sine [default]
      k_mode_sine_mod
      k_mode_cos
      k_mode_cos_mod
      k_mode_square
      k_mode_square_mod
      k_mode_triangle
      k_mode_triangle_mod
      k_mode_ramp
      k_mode_ramp_mod
    k_gain The output level of the oscillator in decibels.
Methods
    set_wavetable(tap_buffer *newbuffer) You can pass a pointer to a tap_buffer object to have the oscillator use your own custom waveform. The oscillator will use the entire length of the buffer you pass it as one period of the wave form.
