tap_downsample
Downsample a signal
This object is downsamples a signal. At the moment it does not use any interpolation. This is intended for applications where a block of objects might be run at a lower sample rate for efficiency. NOTE: the output vector will not be fully used unless the factor is set to "1" (no downsampling).
Attributes
    k_factor Sets the amount of downsampling to a signal. It does this by using the factor to determine how many samples to skip over for each step in the vector. By default this is "1" which equates with no downsampling. A value of "2" would the same cutting the sample rate in half, a value of "4" is cutting the sr to a quarter of the original.

    As a consequence, the output vector will not be fullly utilized. For example, the downsampled output will take only half the vector_size if the factor is set 2. The downsampled output will only use the first 25% of the vector if the factor is set to 4. be sure to pass the new (shorter if you've downsampled) vector_size to the objects that follow this object to get predictable results.
DSP Loop
void dsp_vector_calc(
    sample_vector in,
     short vector_size,
    sample_vector out
)
The entire out vector will probably not be used. The vector_size should indicate the size of the in vector.
