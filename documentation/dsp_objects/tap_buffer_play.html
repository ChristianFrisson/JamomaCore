tap_buffer_play
Play back an audio buffer
This object does not have a self-playing mechanism (at least not at the moment). You will need to drive the playback position manually with another object such as tap_phasor or tap_ramp.

You will also need to specifiy a buffer before using the dsp loop for this object. This may be done as an optional argument to the constructor or by manually setting the attribute after instantiation.
Attributes
    k_mode You can choose the units with which you will drive the playback. The signal input should be in a range matching these units. The attribute should be set with one of the following selectors:
      k_mode_normalized [default]
      k_mode_samples
      k_mode_ms
    k_gain The output level of the oscillator in decibels.
Methods
    set_buffer(tap_buffer *newbuffer) Pass a pointer to a tap_buffer object to have this object reference that buffer.

    Be sure you don't delete the buffer while this object is still playing it! Resizing the buffer while this object is playing should not present any problems.
    get_buffer() Returns a pointer to the tap_buffer object that this object is currently referencing.
DSP Loop
void dsp_vector_calc(
    sample_vector in
    short vector_size,
    sample_vector out
)
Use the "in" sample_vector to pass the location in the buffer to read. The units should match the units specified in the mode attribute.
